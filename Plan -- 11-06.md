**Project**

A computational framework to study how social media ranking algorithms interact with users' cognitive biases to drive disinformation spread and polarization. The methodological core is separating two things the opinion-dynamics literature usually conflates: how agents _update_ opinions (the user) and which information they're _exposed to_ (the algorithm). This first half describes only the user; the algorithm layer builds on top of it later.

**The taxonomy (conceptual backbone)**

User cognitive biases organized by where they act in an information pipeline: input → process → output. Process splits into perception → weighting → integration; weighting splits into four parallel cues (internal compatibility, source value, frequency, salience). Each bias is classified by _injection point_, not cognitive origin. The two boundary surfaces matter because they're where the algorithm will eventually connect: input = what the user receives from the feed, output = the expressed behavior the algorithm observes and trains on.

**The agent model**

A normative Bayesian agent is the baseline, working in log-odds space where updating is just addition:

ℓ ← ℓ + gain · w_compat · w_source · w_novelty · w_salience · LLR(message)

Baseline = every weight at its normative value (gain=1, novelty=0 on repeats, etc.). Each bias is a single signed deformation of one slot — confirmation bias inflates w_compat, illusory truth makes w_novelty > 0 on repeats, asymmetric updating splits gain by sign of the LLR. The classical models (DeGroot, BCM) are _not_ baselines — they're recoverable as deformation profiles (DeGroot double-counts correlated info, i.e. it's the illusory-truth agent).

**Experimental design**

One bias active at a time, confronted against the no-bias baseline, crossed with 4-5 selection rules (random, closest-opinion, most-extreme, engagement/popularity, maybe recency). ~12 biases × 5 rules ≈ 60 cells — tractable. Selection criterion for which biases to implement: per-node coverage first (guarantee every pipeline stage has one), then fill with the cheap ones sharing existing hooks. Output is a "bias atlas": each bias fingerprinted by how its polarization effect changes across selection rules.

**Implementation notes**

Hybrid igraph (structure) + numpy (all numerical state). Pure weight-deformation biases are fully vectorized, negligible cost. Memory-based biases (illusory truth, belief perseverance) need a seen-claims store and run slower — only paid when active. Target N=2,000-5,000, 50-100 replicates per cell, parallelize across cells. Build agent claim-memory from day one since even the baseline needs it for the novelty cue.
